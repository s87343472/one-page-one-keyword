## 核心原则速览

**黄金法则**: 每个页面 = 1个主关键词 + 2-5个次要关键词

- **主关键词**: 定义页面核心主题,唯一优化目标
- **次要关键词**: 相关副主题、同义词、长尾变体,用于完整覆盖主题
- **禁忌**: 多个页面优化同一关键词(导致权重分散)

**为什么有效**: 
- 平均排名第一的页面会在近1000个相关关键词中获得前10排名
- 优化1个精准主关键词,实际流量可达该词搜索量的5-10倍

## 工作流程

### 场景1: 规划关键词布局

1. **创建关键词布局表**
   - 复制 `assets/keyword-layout-template.csv` 作为起点
   - 为每个关键词分配唯一对应页面
   - 记录: URL、页面类型、主/次关键词、搜索量、难度

2. **确保关键词不重复**
   - 检查是否有多个页面针对相同/近似关键词
   - 使用脚本检测: `python scripts/check_keyword_conflicts.py <布局表.csv>`
   - 明确区分父子关键词(如:类目词 vs 产品词)

3. **匹配搜索意图**
   - 分析SERP前10结果的内容类型
   - 确保页面类型与搜索意图一致
   - 参考: `references/seo-principles.md` 中的四大意图类型

### 场景2: 创建新内容前验证

**检查清单**:
- [ ] 此关键词是否已被其他页面使用?
- [ ] 关键词搜索意图与计划的内容类型匹配?
- [ ] 关键词难度与网站当前权重匹配?
- [ ] 是否有足够次要关键词支撑完整主题?
- [ ] URL设计是否包含主关键词?

**快速验证步骤**:
1. 在关键词布局表中搜索该词
2. Google搜索该词,观察SERP前10内容类型
3. 评估难度是否适合当前阶段(新站选KD<20)

### 场景3: 审查现有页面

对每个页面执行:

1. **符合性检查**
   ```
   ✅ 主题明确: 页面是否聚焦单一核心主题?
   ✅ 关键词位置: Title、H1、URL是否包含主关键词?
   ✅ 密度适中: 关键词密度是否在5-10%? (用SEOquake检查)
   ✅ 次要词覆盖: H2-H6是否自然融入次要关键词?
   ✅ 意图匹配: 内容类型是否符合搜索意图?
   ❌ 无堆砌: 是否存在关键词过度重复?
   ❌ 无冲突: 是否与其他页面争夺相同关键词?
   ```

2. **查找问题**
   - 运行冲突检查脚本获取完整报告
   - 查阅 `references/seo-principles.md` 中的常见错误章节
   - 对照检查清单逐项验证

### 场景4: 给出整改建议

#### 问题类型A: 关键词冲突

**诊断**: 多个页面优化相同/相似关键词

**整改方案**:
1. **合并策略** (主题相近时)
   - 选择质量最好的页面作为主页面
   - 整合其他页面内容到主页面
   - 设置301重定向,传递权重

2. **分化策略** (主题不同时)
   - 重新定义各页面主关键词
   - 确保关键词有明确差异
   - 更新关键词布局表

3. **层级策略** (父子关系时)
   - 类目页: 使用父关键词(如"宠物用品")
   - 产品页: 使用子关键词(如"狗玩具")
   - 明确层级,避免竞争

#### 问题类型B: 主题不明确

**诊断**: 一个页面试图覆盖多个不相关主题

**整改方案**:
1. 拆分为多个独立页面,每个页面一个主题
2. 创建支柱页面+关联子页面结构
3. 使用内链连接相关主题

#### 问题类型C: 关键词堆砌

**诊断**: 关键词密度>15%,或不自然重复

**整改方案**:
1. 删除不必要的关键词重复
2. 用同义词替换部分关键词
3. 增加内容长度稀释密度
4. 优先内容可读性,自然使用关键词

#### 问题类型D: 搜索意图不匹配

**诊断**: 内容类型与SERP主流不一致

**整改方案**:
1. 重新分析SERP前10的内容类型
2. 调整页面格式和结构:
   - 信息意图 → 改为教程/指南格式
   - 交易意图 → 改为产品/服务页面
   - 对比意图 → 改为评测/比较文章
3. 或选择搜索意图匹配的新关键词

#### 问题类型E: 关键词难度过高

**诊断**: 新站选择高竞争关键词(KD>40)

**整改方案**:
1. **短期**: 调整为低难度长尾词(KD<20)
2. **中期**: 布局中等难度词(KD 20-40)
3. **长期**: 积累权重后再攻克高难度词
4. 应用"雪崩策略":先做小词获得排名和流量

## 关键指标参考

**快速参考**（详细内容见 `references/seo-principles.md`）：
- 主关键词：1个 | 次要关键词：2-5个 | 密度：5-10%
- 位置优先级：Title > H1 > URL > Meta描述 > 首段 > 子标题
- 难度选择：新站 KD<20 | 成长期 KD 20-40 | 成熟期 KD>40

## 页面类型策略

**快速指南**（各类型详细策略见 `references/seo-principles.md`）：
- **首页**：3-5个行业核心词
- **类目页**：类目核心词（避免与产品页重复）
- **产品页**：产品精准词+属性词
- **博客文章**：信息类/问答类关键词（How-to、指南）
- **解决方案页**：行业应用词+场景词

## 快速诊断问题清单

遇到排名问题时,逐一检查:

1. **关键词层面**
   - [ ] 是否存在关键词冲突?
   - [ ] 主关键词是否明确单一?
   - [ ] 关键词难度是否匹配网站阶段?

2. **内容层面**
   - [ ] 页面主题是否单一清晰?
   - [ ] 关键词密度是否适中?
   - [ ] 是否完整覆盖主题(对比SERP)?
   - [ ] 是否提供独特价值?

3. **技术层面**
   - [ ] Title是否包含主关键词?
   - [ ] H1是否与Title对齐?
   - [ ] URL是否包含关键词?
   - [ ] Meta描述是否吸引点击?

4. **意图层面**
   - [ ] 内容类型是否匹配搜索意图?
   - [ ] 是否符合SERP主流格式?

## 整改优先级建议

**P0 - 立即修复**:
- 关键词严重冲突(3个以上页面竞争同一词)
- 主题完全偏离搜索意图

**P1 - 本周修复**:
- 轻度关键词冲突(2个页面竞争)
- 关键词堆砌严重(密度>20%)
- Title缺少主关键词

**P2 - 本月优化**:
- 次要关键词覆盖不足
- 关键词难度不匹配
- URL未包含关键词

**P3 - 长期优化**:
- 关键词密度微调
- Meta描述优化
- 图片Alt优化

## 记住

- **专注优于分散**: 一个页面一个主题,胜过一个页面多个主题
- **质量优于数量**: 10个精准页面胜过100个泛泛页面
- **自然优于刻意**: 内容可读性永远优先于关键词密度
- **长期优于短期**: 坚持原则6个月+才能看到显著效果

遇到复杂情况时,参考 `references/seo-principles.md` 获取详细指导。
